I am a self-taught Unreal Engine 5 gameplay programmer based in West Yorkshire, focused on C++ systems, Blueprint integration, multiplayer mechanics, UI, and production-relevant gameplay architecture.

Since 2021, I have studied and built a wide range of Unreal projects including inventory systems, combat, AI, multiplayer shooters, GAS RPG mechanics, dedicated server workflows, destructible environments, and UI systems. My current goal is to convert that experience into a clear, verifiable portfolio for junior Unreal gameplay roles.

I build gameplay-focused Unreal Engine systems with emphasis on C++, Blueprint integration, UI, replication, data-driven architecture, and maintainable implementation. This allows me to write my code in a way that exposes features for designers, artists, and other developers without risk to the code base.

In spare time, I enjoy learning, as in programming, there is many different ways of approaching a problem. One way could be using an Array to store a "list" of data. Another way could be to use structs that contain necessary data. Which way is correct? In practice, it depends on project specification, furthermore, what is the intended purpose of the final product? A published game requires both optimised code & reliable/modular framework, so I take on the philosophy of achieving a result that is tailored to that end.
